# 隱私權政策

Highlight Keeper 由使用者自行管理資料，預設不會收集、傳輸或分享任何個人資訊。

## 資料收集與使用
- 儲存內容：使用者自行建立的高亮與筆記（包含網址、標註文字、顏色、標籤、註解）。
- 儲存位置：預設僅存於瀏覽器本機的 `chrome.storage.local`，不會自動上傳伺服器。
- 使用目的：用於顯示與還原標註、搜尋、匯出/匯入。除上述用途外不會另作其他利用。

## 傳輸與分享
- 預設不會將資料傳出裝置，也不會與第三方分享。
- 若使用者手動設定 GitHub Token 與儲存庫並執行「上傳/下載」，備份檔才會透過 GitHub API 傳輸，僅用於備份／還原，不與其他第三方分享。

## 選用的 AI 功能
- 若使用者輸入 OpenAI/Gemini API Key，Key 僅存於本機，僅在使用者要求產生摘要時傳送當前頁面全文與標註給對應的模型 API，用於生成筆記；不另作儲存或分享。

## 權限
- 僅請求執行單一用途所需的權限（storage、activeTab、scripting、內容腳本覆蓋使用者造訪的頁面），用於建立與呈現標註，不用於其他目的。

## 資料保留與刪除
- 使用者可隨時在瀏覽器中刪除擴充功能或清除 `chrome.storage.local`，資料即會移除。GitHub 備份由使用者自行管理，可在儲存庫刪除。
